## CONTEXT

Product: Flyin.to - AI-powered flight search and holiday planning app
Stack: Next.js 15, Supabase (PostgreSQL), SerpAPI, Vercel
Issue: Daily price tracking cron job failing, sending error emails

## REQUIREMENTS GATHERED

Price Tracking Definition:
1. User enables tracking on a project -> first scan sets BASELINE PRICE
2. Daily scans compare current price to baseline
3. If price drops >= 10% from baseline -> email user, RESET baseline to new price
4. If drop < 10% -> do nothing, baseline unchanged
5. Baseline is "rolling" - each alert resets it

User Inactivity Rule:
- 7 days without viewing the specific project -> disable tracking + email
- Activity = viewing the specific tracked project

Constraints:
- One tracked project per user max
- 3 consecutive scan failures -> disable tracking + notify
- Email must succeed before baseline resets
- If email fails twice -> log, alert developer, keep scanning

## ROOT CAUSE IDENTIFIED

The cron job uses `createClient()` from `lib/supabase/server.ts` which:
- Uses ANON KEY with cookie-based session auth
- Has no user context in cron (no cookies)
- RLS policies block all operations (auth.uid() is NULL)

Additional issues:
- Baseline updated on EVERY scan (should only update on alert)
- No email sending implemented
- No inactivity tracking
